# Этап 2: Панель Администратора и Финансы (MVP)

Инструмент управления системой, необходимый модераторам и администраторам для контроля пользователей, транзакций и системных настроек до запуска модуля заказов.

## 1. Разработка Web-интерфейса панели администратора
* **Технологический стек:** Vue 3 (Vite) + [Vuestic Admin Template](https://github.com/epicmaxco/vuestic-admin) & [Vuestic UI](https://ui.vuestic.dev/).
* **Преимущества выбора Vuestic:**
  * Готовая адаптивная панель администратора с боковым меню и поддержкой темной темы.
  * Встроенные компоненты форм, модальных окон и таблиц с высокой производительностью.
  * Интегрированная система маршрутизации (`vue-router`) и готовые макеты страниц авторизации.
* **Авторизация:**
  * Отдельный интерфейс входа для пользователей с ролью `ADMIN`, встроенный в стандартный макет Vuestic.
  * Хранение JWT-токена в Cookie/LocalStorage.
  * Настройка Navigation Guards (`beforeEach`) в `vue-router` для перенаправления неавторизованных пользователей на страницу входа.
  * Проверка роли администратора на уровне эндпоинтов бэкенда при каждом запросе к API панели.

## 2. Управление пользователями
* **Просмотр пользователей:**
  * Интерактивные списки Заказчиков (`CUSTOMER`) и Исполнителей (`EXECUTOR`) на базе компонента `<va-data-table>` с поддержкой фильтрации по номеру телефона, сортировки по дате и серверной пагинации.
  * Текстовый поиск с использованием `<va-input type="search">`.
  * Детальные карточки пользователей на базе `<va-card>`: просмотр профилей (ФИО, адрес, баланс, привязанные данные устройств или геозоны).
* **Модерация и блокировка:**
  * Переключатель статуса пользователя с помощью `<va-switch>` (переключение статуса `status` в таблице `users` между `ACTIVE` и `BANNED`).
  * Кнопка принудительного сброса сессии пользователя и отзыва его JWT-токенов.

## 3. Финансовый модуль MVP
* **Ручное пополнение баланса:**
  * Таблица входящих заявок на пополнение баланса от Заказчиков (`<va-data-table>`).
  * Всплывающее диалоговое окно подтверждения платежа на базе `<va-modal>`. После получения реальных денег на счет администратор подтверждает операцию.
  * Кнопка `<va-button>` для начисления виртуальной валюты (создание транзакции типа `TOP_UP` и обновление баланса пользователя на бэкенде).
* **История транзакций:**
  * Детальный лог балансовых операций (`TOP_UP`, `HOLD`, `PAYMENT`, `FINE`, `REFUND`) с возможностью фильтрации по типу транзакции через выпадающий список `<va-select>`.

## 4. Системные настройки и тарифная сетка
* **Интерфейс управления настройками:**
  * Форма настроек на базе `<va-form>` для редактирования записей в таблице `system_settings`.
  * Поля ввода числовых значений `<va-input type="number">` для:
    * Базовой стоимости стандартного выноса мусора.
    * Повышающих коэффициентов (например, тарифы x2, x3, x8).
    * Размера штрафов для исполнителей за нарушение SLA смены.
  * Сохранение настроек по клику на кнопку `<va-button color="success">` с отправкой `POST /admin/settings`.
