# Что реализовано в текущей версии проекта

На текущем этапе разработки (базовая MVP-версия) реализован следующий функционал:

## 1. Бэкенд (Go)
* **HTTP-сервер:** Настроен роутер `go-chi/v5` для маршрутизации запросов на порту `8080`.
* **Базовые эндпоинты:**
  * `GET /health` — проверка работоспособности сервиса (возвращает `{"status": "ok"}`).
  * `POST /register` — регистрация нового пользователя.
    * Принимает телефон (`phone`) и пароль (`password`).
    * Роль по умолчанию устанавливается в `CUSTOMER`.
    * Хеширует пароль с помощью `bcrypt` (стандартная сложность).
    * Сохраняет пользователя в БД. Возвращает ошибку `409 Conflict`, если телефон уже зарегистрирован.
  * `POST /login` — аутентификация пользователя.
    * Проверяет соответствие телефона и пароля.
    * Возвращает подписанный JWT-токен при успешном входе.
    * В случае ошибок (пользователь не найден, неверный пароль) возвращает общую ошибку `401 Unauthorized` для защиты от перебора.
* **JWT-авторизация:**
  * Токены подписываются алгоритмом `HS256`.
  * Время жизни токена составляет 24 часа.
  * Claims токена содержат: UUID пользователя (`sub`), телефон (`phone`), роль (`role`) и время истечения (`exp`).
  * Секретный ключ передается через переменную окружения `JWT_SECRET`.
* **CORS-фильтр:** Настроено CORS-middleware для обработки запросов со стороны фронтенда (`http://localhost:3000`).
* **База данных:**
  * Интегрирована работа с PostgreSQL без использования ORM (через встроенный пакет `database/sql` и драйвер `github.com/lib/pq`).
  * Реализована функция `waitForDB` для автоматического ожидания готовности БД при старте контейнера бэкенда.

## 2. База данных (PostgreSQL)
Реализована структура таблиц и перечислений (миграции в `backend/migrations`):
* **ENUM-типы:**
  * `role_type` (`CUSTOMER`, `EXECUTOR`, `ADMIN`) — роли пользователей.
  * `status_type` (`ACTIVE`, `BANNED`) — статусы учетных записей.
  * `volume_type` (`STANDARD`, `LARGE`, `CONSTRUCTION`) — типы объемов мусора.
  * `speed_tariff_type` (`REGULAR`, `URGENT`, `ASAP`, `CUSTOM`) — скоростные тарифы.
  * `order_status_type` (`SEARCHING`, `ASSIGNED`, `COMPLETED`, `CANCELED`) — статусы заказов.
  * `shift_status_type` (`ACTIVE`, `COMPLETED`, `PENALIZED`) — статусы смен исполнителей.
  * `transaction_type` (`TOP_UP`, `HOLD`, `PAYMENT`, `REWARD`, `FINE`, `REFUND`) — типы балансовых транзакций.
  * `bid_status_type` (`PENDING`, `ACCEPTED`, `REJECTED`) — статусы ставок в аукционе.
* **Таблицы:**
  * `users` — пользователи (пароли, роли, балансы).
  * `customer_profiles` — профили заказчиков (ФИО, адрес, данные устройства).
  * `executor_profiles` — профили исполнителей (ФИО, геозона).
  * `orders` — заказы по вывозу мусора (статусы, тарифы, суммы).
  * `shifts` — смены исполнителей.
  * `transactions` — финансовые операции пользователей.
  * `bids` — предложения/ставки исполнителей для строительного мусора.
  * `chats` и `messages` — внутренние чаты между участниками сделки.
  * `system_settings` — глобальные тарифные настройки и штрафы.
* **Индексы:** Настроены производительные индексы для связей и поиска.

## 3. Фронтенд (HTML/Vue.js)
* **Single Page Application (SPA):** Простой веб-интерфейс, использующий библиотеку Vue.js 3 (подключается через CDN).
* **Формы авторизации:**
  * Переключение между вкладками Login и Register.
  * Интеграция с эндпоинтами `/login` и `/register` бэкенда.
  * Сохранение полученного JWT-токена в `localStorage`.
  * Перенаправление на страницу `success.html` после успешной авторизации.

## 4. Инфраструктура и DevOps (Docker)
* **Docker Compose:** Настроено окружение для локального запуска из 3 контейнеров:
  1. `db` — контейнер базы данных (PostgreSQL 16) с автоматическим накатыванием файлов инициализации/миграций из `backend/migrations`.
  2. `backend` — контейнер приложения Go (сборка на основе Dockerfile бэкенда).
  3. `frontend` — контейнер на базе Nginx, раздающий статические файлы фронтенда на порту `3000`.
