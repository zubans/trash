# Платформа сервиса по выносу мусора (On-Demand)

## Описание проекта

Данный проект представляет собой маркетплейс услуг по выносу мусора, работающий по модели «по требованию» (on-demand). Платформа связывает заказчиков, нуждающихся в утилизации отходов, с исполнителями, работающими по гибкому сменному графику.

### Ключевые особенности системы
* **Гибкая тарификация:** от стандартных заявок до срочного выноса с динамическим коэффициентом (до x8).
* **Специализированный модуль для строительного мусора:** аукционная модель и торги между заказчиками и исполнителями.
* **Внутренний баланс:** виртуальные кошельки пользователей с механизмом холдирования (заморозки) средств на время выполнения заказа.
* **Операционный контроль:** мониторинг геолокации исполнителей на смене, система автоматических штрафов и автоматический даунгрейд тарифов при нарушении SLA.

### Технологический стек
* **Бэкенд:** Go (Golang)
* **База данных:** PostgreSQL
* **Панель администратора:** Vue.js
* **Клиентские приложения:** Android (нативные/кроссплатформенные)

## 1. Ролевая модель и требования к пользователям

Система разграничивает права и обязанности трех основных типов пользователей:

| Роль | Функции и обязанности | Требования при регистрации |
| :--- | :--- | :--- |
| **Заказчик** | Создание заявок, пополнение баланса, подтверждение выполнения работ. | Номер телефона, пароль (для MVP). |
| **Исполнитель** | Выход на смены, выполнение заявок в закрепленном районе. | Номер телефона, пароль (для MVP). |
| **Администратор** | Модерация, управление экономикой, финансовый контроль, аналитика. | Доступ через веб-интерфейс (Админ-панель). |

### Безопасность и сбор данных
Для минимизации рисков система осуществляет автоматический сбор данных устройства заказчика в фоновом режиме:
* Текущая геолокация (GPS/Wi-Fi/CellID).
* Уникальные идентификаторы устройства (Device ID, IMEI, IP-адрес).
* Данные об операционной системе (тип и версия).

## 2. Авторизация и аутентификация

На текущем этапе реализован базовый механизм регистрации и входа по номеру телефона и паролю.

### Регистрация (`POST /register`)
* **Входные параметры:** номер телефона (`phone`) и пароль (`password`).
* **Поведение:**
  * Пароль хешируется алгоритмом `bcrypt`.
  * В базе данных создается запись в таблице `users` (роль по умолчанию — `CUSTOMER`, статус — `ACTIVE`, баланс — `0`).
  * Если пользователь с таким телефоном уже существует, возвращается ошибка `409 Conflict`.

### Вход / Аутентификация (`POST /login`)
* **Входные параметры:** номер телефона (`phone`) и пароль (`password`).
* **Поведение:**
  * При успешной проверке учётных данных возвращается подписанный JWT-токен.
  * При ошибках (пользователь не найден, неверный пароль) возвращается единый ответ `invalid credentials` (`401 Unauthorized`) для предотвращения перебора номеров.

### JWT-токен
* **Алгоритм подписи:** `HS256`.
* **Время жизни:** 24 часа.
* **Claims:**
  * `sub` — UUID пользователя.
  * `phone` — номер телефона.
  * `role` — роль пользователя.
  * `exp` — время истечения токена.
* **Секретный ключ:** задаётся переменной окружения `JWT_SECRET`.

## 3. Финансовая логика и тарифная сетка

Платформа использует внутреннюю балансовую систему (предоплата).

### Управление балансом
* **Отображение:** Баланс и кнопка пополнения всегда доступны в интерфейсе заказчика.
* **Пополнение:** На этапе MVP реализовано вручную через модератора (администратора).
* **Холдирование:** При создании заказа сумма за него блокируется на счету заказчика. Если средств недостаточно, система предлагает пополнить баланс.
* **Выплата исполнителю:** Начисляется на баланс исполнителя после подтверждения выполнения заказа заказчиком.

### Тарифная сетка
Стоимость услуг изменяется в зависимости от объема и срочности:

| Тип тарифа / Объем | Временные рамки | Коэффициент цены |
| :--- | :--- | :--- |
| **Стандартный (1 пакет)** | Не менее 3 часов ожидания | Базовая цена |
| **Увеличенный (2 пакета)** | Согласно выбранному времени | x2 (настраивается в админке) |
| **Срочный вынос** | В течение 1 часа | x3 |
| **Вынести сейчас** | В течение 15 минут | x8 |
| **Строительный мусор** | До 7 дней (торги) | Договорная цена |

## 4. Механика работы со строительным мусором

Вынос строительного мусора выделен в отдельный бизнес-процесс, основанный на модели торговой площадки (аукциона):

1. **Заявка:** Заказчик создает заявку и обязательно прикрепляет фотографию мусора.
2. **Торги:** Заказчик отправляет запрос без фиксированной цены. Исполнители предлагают свою стоимость, возможен обмен встречными предложениями.
3. **Жизненный цикл:** Заявка активна в течение 7 дней. Если исполнитель не найден, на 8-й день заказ отменяется. Заказчик может продлить или отменить его вручную.
4. **Регламент:** На данный тип заказов не распространяются жесткие требования SLA по времени прибытия.

## 5. Операционная деятельность исполнителей

Работа исполнителей регламентирована для обеспечения стабильного покрытия сервиса:

* **Управление сменами:** Исполнитель выбирает длительность смены (1, 3 или 5 часов) и конкретное время.
* **Геофиксация:** Во время смены исполнитель обязан находиться в пределах своей рабочей зоны (контролируется по GPS).
* **Система штрафов:** Самовольный уход со смены или выход за пределы рабочей зоны влечет автоматическое списание динамического штрафа с баланса исполнителя.
* **Очередь заказов:** Заявки распределяются последовательно; исполнитель обязан принимать и исполнять поступающие запросы.

## 6. Контроль качества и коммуникации

### Подтверждение выполнения (Proof of Work)
В целях упрощения MVP-версии подтверждение возлагается на заказчика. В приложении предусмотрена кнопка «Мусор забран». Только после её нажатия заказ считается выполненным, а заблокированные средства переводятся исполнителю.

### Нарушение SLA
Если исполнитель опаздывает на заказ по тарифу «Вынести сейчас» (лимит 15 минут):
* Происходит автоматический даунгрейд тарифа до стандартного.
* Разница в стоимости моментально возвращается на баланс заказчика.

### Внутренние коммуникации
Для уточнения деталей заказа предусмотрен встроенный чат:
* Доступен только между заказчиком и назначенным исполнителем.
* Существует строго в рамках конкретного заказа.
* Автоматически переходит в режим «только чтение» после завершения или отмены заказа.

## 7. Панель администратора (Модерация)

Веб-интерфейс администратора (Vue.js) предоставляет следующие возможности:

* **Управление пользователями:** Просмотр профилей пользователей, блокировка/разблокировка аккаунтов.
* **Мониторинг заказов:** Отслеживание статусов заказов в реальном времени.
* **Финансовое управление:** Ручное начисление баланса пользователям, настройка базовых цен, повышающих коэффициентов и штрафов.
* **Аналитика:** Статистика по популярным районам, тарифам и обороту средств.
